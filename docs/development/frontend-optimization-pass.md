# BranchWhisper 前端一轮优化方案

## 目标

这轮优化把“检测中心”拆回各功能页，让用户在配置、素材、服务、接入、记忆页面就能完成最小可用回路测试；同时压缩页面空白、修复配置面板滚动体验，并统一浅色主题下的状态样式。

## 一次完成范围

1. 删除独立检测页
   - 前端路由和导航不再出现“检测”页面。
   - 删除 `DiagnosticsPage.vue` 和 `diagnostics.css`。
   - 保留 `/api/diagnostics/*` 后端接口，作为功能页内联检测的后端能力。

2. 分布式最小检测回路
   - 配置页：对话 API、本地 ASR/LLM/TTS/后端、Vision API、联网工具 Provider、主动消息。
   - 素材页：素材识图 API、表情策略匹配。
   - 服务页：本地服务 health 聚合检测。
   - 接入页：文本回复、语音发送、素材发送三条微信链路。
   - 记忆页：记忆 admission 测试，确认一句话是否会入库。

3. 配置页体验
   - 打开的配置面板按内容自适应高度，并设置最大高度。
   - 打开面板时锁住 body 滚动。
   - 面板内部使用 `overscroll-behavior: contain`，滚轮不再带动外层页面。
   - 工具 Provider 卡片加入独立测试条，避免配置项和检测结果挤在一起。

4. 空白与密度
   - 记忆页左栏不再用剩余高度强行撑开。
   - 配置页短面板不再产生大块空底。
   - 素材页把检测条放在配置条与统计之间，填补原本割裂的间距。
   - 接入页测试结果从大块 `pre` 改成可折叠感更轻的内联检测条。

5. 主题修复
   - 浅色主题下 `status-pill`、顶部状态和侧栏 pipeline 使用明确的浅色背景。
   - 避免“待机”状态块继承深色背景。

## 验收标准

1. `npm --prefix frontend run check` 通过。
2. 访问 `/app/`、`/app/settings`、`/app/assets`、`/app/services`、`/app/integrations`、`/app/memory` 无运行时错误。
3. 顶部导航没有“检测”入口，直接访问 `/app/diagnostics` 不应再作为正式入口使用。
4. 配置页打开任意高级面板后：
   - 面板内部可以滚动。
   - 鼠标滚轮不会带动背景页面。
   - 短内容面板没有明显的大块空底。
5. 每个功能页至少有一个真实 API 调用测试按钮，失败时能显示可复制的诊断结果。

## 后续优化队列

1. 统一检测状态模型
   - 把 `InlineProbe` 的状态、复制、JSON 格式化抽成 composable。
   - 让所有页面的检测结果都能持久化最近一次状态。

2. 对话与微信链路
   - 为入站消息加幂等 key，避免同一条微信消息重复回复。
   - 文本、语音、表情三类发送使用统一 trace id，前端展示每阶段耗时。
   - 表情发送增加更严格的语境阈值和最近回复去重。

3. 记忆调用可视化
   - 对话页展示本轮检索到的记忆摘要。
   - 记忆页增加“命中测试”：输入一句话，显示会召回哪些记忆。

4. 配置保存反馈
   - 保存后逐项校验配置回显。
   - 对 API Key、路径、端口字段给出即时状态提示。

5. 页面信息架构
   - 配置页继续把高频项留在首页，低频项进面板。
   - 服务页把“路径不存在、命令不一致、健康失败”做成可展开诊断。
   - 素材页增加“最近为什么选中这个表情”的解释面板。
